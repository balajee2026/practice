Set up SSH key-based authentication between two Linux severs. 

Test-2 (Server)
===============
Install SSH:
sudo apt update
sudo apt install openssh-server

Open /etc/ssh/sshd_config and add/edit:
PermitRootLogin no  
PasswordAuthentication yes  
PubkeyAuthentication yes

Restart SSH server: sudo systemctl restart sshd
Allow SSH traffic in the firewall: sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT

Test-1 (Client)
======
Try connecting to Test2 using SSH (SSH server):
ssh username@server_ip
e.g. ssh atul@192.168.0.109
If successful, check ip addr and then exit to return to Test-1 machine


Generate key pair on Test-1:
ssh-keygen -t rsa

Follow the prompts to generate the key pair,
Two files will be created in ~/.ssh/ directory named id_rsa (private key) and id_rsa.pub (public key)

Copy the public key to the server by running this command:
ssh-copy-id user@server_ip
Replace user with user name and server_ip with IP address of Test-2 machine


Verify on Test-2 machine by doing an ls of ~/.ssh directory
We should see a file named authorized_keys there

If this does not work, copy the file manually to Test-2 machine and rename as as ~/.ssh/authorized_keys
You will be asked for the password for Test-2 machine

Hash a File:

openssl dgst-sha256 -out file1.txt.sha256 file1.txt

Hash file1.txt and store its hash in file1.txt.sha256

Make changes to file1.txt and recalculate the hash in a different file:
openssl dgst-sha256 -out file1-mod.txt.sha256 file1.txt

Compare the contents of the two .sha256 files
diff file1.txt.sha256 file1-mod.txt.sha256



Digital Signature:

Generate private key file: 
openssl genpkey-algorithm   RSA   -out   keyfile.pem-aes256

Derive public key from the private key: 
openssl rsa -in keyfile.pem -pubout -out public_key.pem 

Digital signature: 
openssl dgst-sha256 -sign keyfile.pem -out file1.txt.sha256.sig      file1.txt

Verify signature:
openssl dgst-sha256 -verify public_key.pem -signature   file1.txt.sha256.sig file1.txt


CSR and Digital Certificates: 

Step 1: Create public-private key pair (RSA) and save in a file:

openssl genpkey-algorithm RSA -out alice.key

Step 2: Extract the public key into a separate file:

openssl pkey -pubout-in alice.key-out alice.pub

Step 3: View the keys:

openssl pkey -in alice.key -text -noout
openssl pkey -pubin -in alice.pub -text -noout

Step 4: Create a Certificate Signing Request (CSR):

openssl req -new -key alice.key -out alice.csr

Step 5: Create a digital certificate using the CSR:

Generate root CA key pair:
openssl genpkey-algorithm    RSA    -out    root.key

Generate root CA Certificate Signing Request (CSR):
openssl req    -new    -key    root.key-out    root.csr

Self-sign the Root CA certificate:
openssl x509 -req -in root.csr -signkey root.key -out root.crt -days 3650

View the file in a readable format:
openssl x509 -in root.crt -text -noout

Sign Alice CSR with Root CA:
openssl x509 -req -in alice.csr -CA root.crt -CAkey root.key -CAcreateserial -out alice.crt -days 365

View the file in a readable format:
openssl x509 -in alice.crt -text -noout

Set Up PKI Infrastructure Using OpenSSL:
Establish a Public Key Infrastructure (PKI) with OpenSSL, involving the creation of a Root Certificate Authority (CA) and a subordinate (Sub) CA.
Issue digital certificates from the Sub CA for various purposes.


Root CA Certificate:

Generate root CA private key:
openssl genpkey -algorithm RSA -out root.key

Generate root CA Certificate Signing Request (CSR):
openssl req -new -key root.key-out root.csr

Self-sign the Root CA certificate:
openssl x509 -req -in root.csr -signkey root.key -out root.crt -days 3650

View the file in a readable format:
openssl x509 -in root.crt -text -noout


nano sub.ext
basicConstraints=critical,CA:TRUE,pathlen:0
keyUsage=critical,keyCertSign,cRLSign
subjectKeyIdentifier=hash
authorityKeyIdentifier=keyid,issuer



Sub CA Certificate
Generate sub CA private key:
openssl genpkey -algorithm RSA -out sub.key

Generate sub CA Certificate Signing Request (CSR):
openssl req -new -key sub.key-out sub.csr

Sign the sub CA CSR with Root CA:
openssl x509 -req -in sub.csr-CA root.crt -CAkey root.key -CAcreateserial -out sub.crt -days 1825 -extfile sub.ext

View the file in a readable format:
openssl x509 -in sub.crt -text -noout

Generate some server’s private key:

Diffie-Hellman Key Exchange:

Generate Diffie-Hellman global parameters:
openssl genpkey -genparam -algorithm DH -out dhp.pem
Display the global parameters in encoded form
cat dhp.pem
Display the global parameters in text form
openssl pkeyparam -in dhp.pem -text


Each User Generates Key Pair:

openssl genpkey -paramfile dhp.pem -out dhkey1.pem
openssl pkey -in dhkey1.pem -text -noout
openssl genpkey -paramfile dhp.pem -out dhkey2.pem
openssl pkey -in dhkey2.pem -text -noout


Users Exchange Public Keys:

User 1: Extract the public key into file dhpub1.pem 
openssl pkey -in dhkey1.pem -pubout -out dhpub1.pem
openssl pkey -pubin -in dhpub1.pem -text

User 2: Extract the public key into file dhpub2.pem
openssl pkey -in dhkey2.pem -pubout -out dhpub2.pem
openssl pkey -pubin -in dhpub2.pem -text


Independently Derive Shared Secret/Symmetric Key:
User 1:
openssl pkeyutl -derive -inkey dhkey1.pem -peerkey dhpub2.pem -out secret1.bin
User2:
openssl pkeyutl -derive -inkey dhkey2.pem -peerkey dhpub1.pem -out secret2.bin

Confirm that two keys are same:
diff secret1.bin secret2.bin
